What is GitHub?
###############
GitHub is a web-based platform that is used to store, manage, and share source code using the Git version control system. It allows developers to upload their code into repositories, track changes made by different team members, and collaborate efficiently on the same project. GitHub provides features like branching, merging, pull requests, and issue tracking, which help in managing development work, reviewing code, and fixing bugs in an organized way. It is widely used in software development to maintain code history, support team collaboration, and ensure smooth project version control.




################################
Difference Between Git & GitHub?
################################
Git is a distributed version control system that is used to track changes in source code, manage different versions, and allow developers to work on a project locally without needing an internet connection. GitHub is a web-based platform that uses Git and provides a central place to store Git repositories online, enabling team collaboration, code sharing, and project management through features like pull requests, code reviews, and issue tracking. Git focuses on version control at the local level, while GitHub focuses on collaboration and repository hosting at the central level.





######################
Why we require GitHub?
######################
GitHub is required to store code in a central and secure place where multiple developers can access and work on the same project. It helps teams collaborate easily by allowing code sharing, version tracking, and reviewing changes through pull requests. GitHub also maintains complete code history, helps in managing bugs and tasks using issues, and supports smooth integration with tools for testing and deployment, which makes development faster, more organized, and more reliable.

##########################
What is working directory?
##########################
The working directory is the local folder on a developerâ€™s system where the project files are stored and actively modified. It contains the current version of the code that a developer is working on, and any changes made in this directory are not tracked by Git until they are added to the staging area. The working directory allows developers to edit, test, and review code before committing it to the Git repository.





#################
What is staging ?
#################
Staging is an activity where I am selecting only those files which I want to take it as a backup for us is called as Staging.

Staging is the intermediate area in Git where selected changes from the working directory are prepared before committing them to the repository. It allows developers to choose which files or changes should be included in the next commit, helping to organize and control code updates. By using the staging area, developers can review and verify changes before permanently saving them in the Git version history.

##################
What is branches ?
##################
It allows developers to work on new features, bug fixes, or changes independently without affecting the main codebase. Each branch contains its own set of commits and points to a specific version of the code, making it easy to develop, test, and modify features in isolation. Once the work on a branch is completed and reviewed, it can be merged back into the main branch, which helps teams manage parallel development safely and keep the code stable.
